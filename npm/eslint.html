# eslint-config-sd

TIP

前端 JavaScript TypeScript Node 规范

提供了多套配置文件以支持 JavaScript、TypeScript、React、Vue、Node.js 等多种项目类型。

# JavaScript 项目 - eslint-config-sd

针对未使用 React 或 Vue 的原生 JavaScript 项目，使用 ESLint 原生规则和 eslint-plugin-importopen in new window 规则，使用 @Babel/eslint-parseropen in new window 作为 parser，是本包的默认配置。

# 依赖

  • @Babel/coreopen in new window@^7.16.0
  • @Babel/eslint-parseropen in new window@^7.16.3
  • eslint-plugin-importopen in new window@^2.25.3

# 安装

npm i -D eslint-config-sd @babel/core @babel/eslint-parser eslint-plugin-import

# 配置

{
  "extends": ["eslint-config-sd"]
}

# JavaScript + React 项目 - eslint-config-sd/React

针对 JS React 项目，继承了默认配置，并启用了 eslint-plugin-Reactopen in new window 和 eslint-plugin-React-hooksopen in new window 的规则。

# 依赖

  • @Babel/coreopen in new window@^7.16.0
  • @Babel/eslint-parseropen in new window@^7.16.3
  • eslint-plugin-importopen in new window@^2.25.3
  • eslint-plugin-Reactopen in new window@^7.17.0
  • eslint-plugin-React-hooksopen in new window@^4.2.0
  • eslint-plugin-jsx-a11yopen in new window@^6.3.1（可选）

# 安装

npm i -D eslint-config-sd @babel/core @babel/eslint-parser eslint-plugin-import eslint-plugin-react eslint-plugin-react-hooks

# 配置

{
  "extends": ["eslint-config-sd/react"]
}

如果需要无障碍能力：

npm i -D eslint-plugin-jsx-a11y

{
  "extends": ["eslint-config-sd/react", "eslint-config-sd/jsx-a11y"]
}

# TypeScript 项目 - eslint-config-sd/typescript

针对未使用 React 或 Vue 的 TypeScript 项目，继承了默认配置，并启用了 @typescript-eslint/eslint-pluginopen in new window 插件的规则，使用 @typescript-eslint/parseropen in new window 作为 parser。

# 依赖

  • @typescript-eslint/parseropen in new window@^5.0.0
  • @typescript-eslint/eslint-pluginopen in new window@^5.0.0
  • eslint-plugin-importopen in new window@^2.25.3
  • eslint-import-resolver-typescriptopen in new window@2

# 安装

npm i -D eslint-config-sd @typescript-eslint/parser @typescript-eslint/eslint-plugin eslint-plugin-import eslint-import-resolver-typescript

# 配置

{
  "extends": ["eslint-config-sd/typescript"]
}

需保证项目已安装 typescript 依赖，另外如果项目的 TS 配置文件不是 ./tsconfig.json，则需要设置 .eslintrc 中的 parserOptions.projectopen in new window 字段 ，例如：

{
  "extends": "eslint-config-sd/typescript",
  "parserOptions": {
    "project": "./tsconfig.eslint.json"
  }
}

# 配合 Prettier 使用

如果你的项目使用 Prettieropen in new window 进行代码格式化，本包的一些规则可能会跟 Prettier 格式化结果有冲突，例如这条规则open in new window。为了避免冲突，你需要手动安装 eslint-config-prettieropen in new window 和 eslint-plugin-prettieropen in new window：

# 安装

npm install --save-dev eslint-config-prettier eslint-plugin-prettier

# 配置

并修改 .eslintrc 的 extends 配置，增加 prettier，如下（以 TS React 项目为例）：

{
  "extends": ["eslint-config-sd/typescript/react", "prettier"]
}

了解更多请阅读 Prettier - Integrating with Lintersopen in new window。

# 将风格问题降级

为了保证一致的编码风格，本包中大量风格相关的规则被设为了 error 级别，以引起开发者的足够重视。如果你觉得风格问题不足以是 error 级别（有些用户根据 ESLint error 进行流程卡点），本包还提供了一套名为 'essential' 的配置文件，这套配置将所有风格问题降级为 warn 级别，仅将必要问题报告为 error，引用方式为在相应配置的 eslint-config-sd 后面加上 /essential，如对 JS React 项目为 eslint-config-sd/essential/react、对 TS Vue 项目为 eslint-config-sd/essential/typescript/vue

# 了解更多

  • 如果你对 ESLint 还不熟悉，可以阅读官网的 Getting Startedopen in new window 快速入门。
  • 了解如何为 IDE 配置 ESLint，可以参考官网的 Integrationsopen in new window。
  • 了解如何在继承本包的基础上对项目 ESLint 进行个性化配置，可参考官网的 Configuring ESLintopen in new window。下面简介下 ESLint 配置中的几个常用字段：
    • extends: 继承一组规则集。"extends": "eslint-config-sd", 表示继承本包定义的规则配置。
    • rules: 配置规则，这里定义的规则会覆盖 extends 的规则。如果觉得本包开启的某条规则过于严格，你可以暂时在这里将其关闭。
    • parser: 设置 ESLint 的解析器。ESLint 使用 espree 作为默认的解析器，可以通过这个参数指定其他的解析器。比如指定为 @Babel/eslint-parseropen in new window，以解析 Babel 支持但 ESLint 默认解析器不支持的语法（本包不同配置文件使用的解析器可在简介表格中的「依赖 parser」一列查看）。
    • globals: 指定代码中可能用到的全局变量，以免全局变量被 no-undefopen in new window 规则报错。
    • env: 指定代码的运行环境，每个环境预定义了一组对应的全局变量，本包已开启的环境有 browser、node、jQuery、ES6 及几个测试框架的环境。
  • 了解常用的 ESLint 命令，如 --fix、--ext，可参考官网的 Command Line Interfaceopen in new window。
